搞定String对象引用的比较

Posted on 2017-10-02 Edited on 2018-11-07 In Java基础

前言

最近刷题总是刷到跟String类型比较的题目，提着刀过来写下了这篇博客
Tips:

  • **==**比较的是两个引用的地址
  • String引用赋值String字面量不会调用String的任何方法，JVM会直接在常量池中创建String的字符串对象
  • String字面量的相加不会产生新对象，例如”a”+”bc”和”abc”是完全相等的，因为JVM在加载class时识别到字面量会自动优化

String类型的创建

一个问题

String s;
String s = null;
String s = "";
String s = new String();

请问上面这四条语句有啥区别？嗯，头有点晕，我们一个一个来分析。

  • String s;
    该语句表示只是声明了一个引用变量在栈中,但是并没有初始化引用,所以对变量s的任何操作(除了初始化赋值外) 都将引发异常。
  • String s = null;
    表示未申请任何内存资源，即些语句表示声明了一个引用变量并初始化引用,但是该引用没有指向任何对象.但可以把它作为参数传递或其它使用,但是不能调用它作为对象的方法
    关于null的更多细节，可以看看关于Java中null的十点详解
  • String s = “”;
    表示在栈中创建引用s，并创建一个字符串长度为0的字符串对象，该对象位于方法区的常量池，引用s指向常量池。
  • String s = new String();
    表示在栈中创建引用s，并创建一个字符串长度为0的字符串对象，该对象位于堆内存，引用s指向堆内存地址，同时值对象存储在常量池。它不等于null，因为它在初始化时自动赋了值： 所以这条语句等同于
      String s = new String("");

没有生成String对象的方法

  • String类的toString()方法

    输出：

      true

    源码：

      public String toString() {
      return this;
  }

    因为String类重写的toString()方法会返回自身，其他类的toString()方法都生成了新对象。

  • String.valueOf(Object o);

    输出：

      true true

    源码：

      public static String valueOf(Object obj) {
      return (obj == null) ? "null" : obj.toString();
  }
  public String toString() {
      return this;
  }

    可以看到，一顿操作以后，还是把自己送出去了zZ…
    需要注意的是，如果是Object类，那只有赋值为字面量字符串时才能相等，因为o自动向下转型为String类型，s引用的就是o在常量池生成的对象~

  • String类的intern()方法

    输出：

      false true

    源码：

      public native String intern();

    String类的intern方法是一个native方法，它返回对象的字符串常量，上面的s2返回的就是”123”，s1没有变化，前提是常量池已经存在该对象了；
    如果不存在该对象：

    输出：

      true true

    我们发现s本来指向的是堆内存的引用，竟然也指向了常量池的”123”，这就是intern()方法的作用。

    所以，除了这几个特殊的方法之外，其他方法例如什么replace()、subString()、trim()、copeValueOf()等等都生成了新对象，都为false.
