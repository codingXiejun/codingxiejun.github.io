学习WebService

发表于 2018-12-11 分类于 JavaWeb

前言

前两天面试时有个笔试题问的是WebService的概念，虽然经常听到这个词但是概念很模糊，花点时间来学习一下，补补漏洞。

什么是WebService

假如有一个场景，我们的网页需要显示当前天气，我们自己能用JS代码能写出来吗？明显是不可能的，它的数据是依赖数据库分析出来的，甚至需要卫星探测，我们个人建站是不可能搞这么一个数据库的吧。
但是有很多天气预报的网站，我们不可能把整个网页的代码都复制到我们的网页上去，我们需要的只是一个接口来提供对应天气的数据，于是乎WebService就产生了。
看名字就可以看出来，WebService是一个类似Web服务器的东西，它提供的是一种类似API接口的服务，例如我的博客中嵌入了例如网易云插件、天气插件等程序，商户的POS转账也是运行在银行的服务器上等。
百度百科对它的定义:

Web service是一个平台独立的，低耦合的，自包含的、基于可编程的web的应用程序，可使用开放的XML（标准通用标记语言下的一个子集）标准来描述、发布、发现、协调和配置这些应用程序，用于开发分布式的互操作的应用程序。
Web Service技术， 能使得运行在不同机器上的不同应用无须借助附加的、专门的第三方软件或硬件， 就可相互交换数据或集成。依据Web Service规范实施的应用之间， 无论它们所使用的语言、 平台或内部协议是什么， 都可以相互交换数据。

如何使用WebService

我们知道写网站有很多语言可以完成，比如Java、.net、php等等，那么部署在Web服务器上的服务器也就是WebSerice怎么能够就让我们不同的平台都能够调用呢？
我们知道java、.net这样的平台他们语言的基本数据类型、复杂数据类型就可能不一样，那么怎么能够实现调用的呢？
无论是Java、.net、PHP等等的平台，只要是网页开发都是可以通过http协议来进行通信的，并且返回的数据要是通用的话，那么我们早就学过这样的一种技术【XML】
所以webservice实际上就是Http+XML

WebService三要素

  • SOAP (Simple Object Access Protocol)：简易对象访问协议，soap用来描述传递信息的格式。
  • WSDL (WebServices Description Language)：Web服务描述语言，用来描述如何访问具体的接口。
  • UDDI (Universal Description Discovery and Integration)：通用描述、发现及整合，用来管理、分发、查询webService。
